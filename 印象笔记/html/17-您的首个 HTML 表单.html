17-您的首个 HTML 表单

创建我的第一个表单

任务目标：
  1. 熟悉HTML表单是什么，它们被用来做什么，如何设计它们，以及简单情况下需要的基本HTML元素。



HTML表单是用户和web站点或应用程序之间交互的主要内容之一，由一个或多个小部件（控件）组成：文本字段(单行或多行)、选择框、按钮、复选框或单选按钮。

大多数情况下，数据被发送到web服务器，但是web页面也可以自己拦截它并使用它。

表单越大，失去用户的风险就越大。保持简单，保持专注:只要求必要的数据。

HTML表单以一个<form>元素开始：

<form action="/my-handling-form-page" method="post">

</form>

<form>的两个属性（最佳做法）： 
  1. action 属性定义：在提交表单时,收集的数据送哪(URL)去处理。
  2. method 属性定义:发送数据的HTTP方法(“get”或“post”)。


label（标签控件），input（输入框）

1、<label>和<input>：单行文本字段。<textarea>：多行文本字段。

<label for="name">Name:</label> <input type="text" id="name" name="user_name">

label与input等表单控件是共生的！

为什么要给<label>加上for属性

这个属性引用相应的表单控件，允许用户单击标签以激活相应的表单控件。（即通过for属性与表单控件的id相关联）

如果点击label元素内的文本，就会触发此控件即当用户点击该标签时，浏览器就会自动将焦点转到和标签相关的表单控件上。


label标签通常是写在表单（form）内的，他与一个普通的span最大的不同，就是可以和表单元素配对，比如文本框，单选框，复选框。而配对的方法，就是通过label的for属性。
比如

<input type="text" name="" id="text" /><label for="text">一个label，配文本框</label><br/>  

<input id="User3" type="radio" name="gender" value="3">  

<span for="User3">一个span</span>  

<label for="User3">label配radio</label> 

测试以上代码，你会发现点击“配文本框”的label时，它前面的input:text标签获得焦点了（可输入）！而点击“配radio”时，它前面的单选框radio就被选中了；但点击“一个span”时，单选框不会有什么反应，虽然它们离得更近。
这些只是因为label的for属性。它为label指定了一个目标，一般是一个form表单内元素的ID，这样就可以把这个label与该ID元素关联起来，当点击label时，就可以让该ID也获得焦点！

input标签

<input> 标签是一个空元素，不能包含子元素。<textarea>不是一个空元素！

<input>的type属性：输入的文本字段类型

<input type="text" id="name" />
<input type="email" id="mail" />

<input>的value 属性（搞特殊）：默认值


注意：<input type="checkbox"> 和 <input type="radio"> 中必须设置 value 属性。

<textarea>标签

<textarea id="msg"></textarea>

<textarea>不需要value 属性，<textarea>的默认值只需在<textarea>元素的开始和结束标记之间放置默认值

<textarea>by default this element is filled with this text</textarea>


2、<button> 元素

<div class="button"><button type="submit">Send your message</button></div>

<button>元素的 type属性，它接受三个值中的一个:submit, reset或者 button。
单击 submit 按钮 发送表单的数据到<form>元素的action 属性所定义的网页。
单击 reset 按钮 将所有表单控件重新设置为它们的默认值。用户体验的角度来看，这是一种糟糕的做法。
单击 button 按钮……不会发生任何事！这听起来很傻，但是用JavaScript构建定制按钮非常有用。


3、向您的web服务器发送表单数据

为我们的数据（input）提供一个name名称：
  1. 浏览器端能告诉浏览器哪个名称提供哪些数据。
  2. 服务器端可以让服务器按名称处理每个数据块。

<form action="/my-handling-form-page" method="post">
<div>
<label for="name">Name:</label>
<input type="text" id="name" name="user_name" />
</div>
<div>
<label for="mail">E-mail:</label>
<input type="email" id="mail" name="user_email" />
</div>
<div>
<label for="msg">Message:</label>
<textarea id="msg" name="user_message"></textarea>
</div>

...

表单会发送三个已命名的数据块 "user_name", "user_email", 和 "user_message"。这些数据将使用HTTP POST 方法,把信息发送到URL为 "/my-handling-form-page"目录下。
服务器端，位于URL"/my-handling-form-page" 上的脚本将接收的数据作为HTTP请求中包含的3个键/值项的列表（应该是Json值，就像user_name=...，user_email=...，user_message=...）。
