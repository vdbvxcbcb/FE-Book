3-HTML 文本基础

HTML 文本基础

任务目的：
  1. 学习如何用(段落、标题、列表、强调、引用)来标记基本的文本页面，赋予页面结构和意义。

基础：


标题和段落

<p>元素（paragraph）

<p>I am a paragraph, oh yes I am.</p>

<hx>元素（heading文章标题）

<h1>I am the title of the story.</h1>

六个标题元素标签— <h1>, <h2>, <h3>, <h4>, <h5>, 和 <h6>。<h1>表示一级主标题（the main heading），<h2>表示二级子标题（subheadings），<h3>表示三级子标题（sub-subheadings）。

最佳做法：
  1. 一个页面使用一次<h1> — 这是顶级标题。
  2. 以正确的顺序使用标题。
  3. 一个页面使用不超过三个文章标题。



为什么我们需要结构化?

没有结构，文本显示为一整块！




没有元素给内容结构的后果：
  1. 浏览器不知道什么是标题，什么是段落。
  2. 用户不能在几秒内看到一些有用的内容，很可能会沮丧地离开。
  3. 屏幕阅读器用户无法快速找到他们需要的信息，被迫听到整个文档的大声朗读。
  4. 影响网页搜索排名。
  5. 无法使用CSS样式化和JavaScript做一些有趣的事情。

屏幕阅读器用户：色盲患者大概占了世界人口的 8%，盲人和视障人士占世界人口的4-5％左右（2012年全球有2.85亿人, 总人口约70亿）。

语义：当我们看到什么，我们就会知道它代表什么。例如： 我们知道红色交通灯表示“停止”，绿色交通灯表示”通行“。

语义（含义）化：使用正确的标签表达HTML元素内容的意思或作用。编写HTML代码是用来表示填入的数据（内容），不是用来展示样式的，展示样式是CSS唯一的责任。

语义化（结构化）的好处：
  1. 提高搜索排名（SEO）。
  2. 帮助屏幕阅读器用户浏览页面。
  3. 即使有很多class的div也很容易找到有意义的代码块。
  4. 方便元素命名。


语义化作用总结
    ● 使用正确的标签表达HTML元素内容的意思或作用
    ● 提升代码的可读性、可维护性，方便元素命名。
    ● 提高搜索排名（利于SEO搜索引擎的优化）
    ● 提升无障碍性


在碰到要使用哪个标签时，问下自己，“哪些元素最能描述/表示我要填充的数据？”例如：它是一个数据列表吗？ 有序的，无序的？
它是需要说明标题（caption）还是需要说明文字（figure）的图像吗？它是一章有标题（section）的文章（article）还是相关信息的旁白（aside）？

<span>标签是没有语义的内联元素，用来设置单独的样式。

例子：“提示”文字使用粗体的橘红色

HTML：

<p class="tip"><span>提示：</span>... ... ...</p>

CSS：
p.tip span {
        font-weight:bold;
        color:#ff9955;
        }


列表 Lists

无序列表（Unordered lists ）

ul-li是项目没有前后顺序的列表，顺序并不重要。ul-li在网页中显示的默认样式一般为：每项li前都自带一个实心圆点。


有序列表（Ordered lists）

ol-li是项目有前后顺序的列表，ol-li在网页中显示的默认样式一般为：每项<li>前都自带一个序号，序号默认从1开始.。


嵌套列表（Nesting lists）

让一些子项目列在更高级的项目之下

<ol>
  <li>Remove the skin from the garlic, and chop coarsely.</li>
  <li>Remove all the seeds and stalk from the pepper, and chop coarsely.</li>
  <li>Add all the ingredients into a food processor.</li>
  <li>Process all the ingredients into a paste.
    <ul>
      <li>If you want a coarse "chunky" humous, process it for a short time.</li>
      <li>If you want a smooth humous, process it for a longer time.</li>
    </ul>
  </li>
</ol>





强调和重要性（都表示强调）

强调（Emphasis）

<em>（emphasis）元素（斜体强调）

在口语方面，往往用重音强调来巧妙地改变我们想要说的意思。在写作方面，将文字写成斜体字来达到强调的效果。

不能使用这个标签来获得斜体样式。获得斜体样式应该使用<span>元素和一些CSS，或者<i>元素。

<p>I am <em>glad</em> you weren't <em>late</em>.</p>


重要性（Strong importance）

<strong> (strong importance) 元素（重点强调）

强调重要的词，在口语方面我们往往用重音强调，在文字中用粗体字来达到强调的效果。

不能使用这个标签来获得粗体样式。获得粗体样式应该使用<span>元素和一些CSS，或者<b> 元素 。

<p>This liquid is <strong>highly toxic</strong>.</p>

<p>I am counting on you. <strong>Do not</strong> be late!</p>





斜体字、粗体字、下划线


<i>,<b>,  和 <u> 出现于人们在文本中使用粗体、斜体、下划线但CSS仍然不被完全支持的时期。这样的元素仅仅影响表象而且没有语义，被称为表象元素（presentational elements）并且不应该再被使用。

最佳做法：

只在没有更适合的语义元素表示时使用。

例如：
  • 使用 <em> 表示强调或重读。
  • 使用 <strong> 表示重要性。
  • 使用 <mark> 表示相关性。标记黄色高亮效果。（HTML5）
  • 使用 <cite> 标记著作名，如一本书、剧本或是一首歌。
  • 使用 <dfn> 标记术语的定义实例。

使用<b>,<i>,<u> 来传达传统意义上的粗体，斜体或下划线是合适的，没有其他元素更适合这样用了。

<i>,<b>,   <u>现在的语义：

●<i>现在的语义：表达传统意义上的斜体，表示外文词汇或科技术语。

●<b>现在的语义：表达传统意义上的粗体，表示关键字和产品名称。

●<u>现在的语义还是添加下划线。

注意：人们很容易把下划线和超链接联系起来！

保持可访问性（更好地让搜索引擎理解）思维至关重要。

<i>（italic）以前废弃的语义：斜体   非语义方式让文本显示斜体：使用CSS的 font:italic;     

<b> (bold)  以前废弃的语义：粗体     非语义方式让文本显示粗体：使用CSS 的 font-weight:bold;    
<span style="font-weight:bold;">Today's Special</span>

<u> (underline)下划线    非语义方式给文本添加下划线： <span> 元素和CSS的 text-decoration:underline ;排版。  
<span style="text-decoration:underline;">Today's Special</span>
