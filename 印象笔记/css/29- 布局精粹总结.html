29- 布局精粹总结









水平居中布局

1、文本/行内元素/行内块级元素

.parent {
    text-align: center;
}

2、单个块级元素

子级 display: block 或 table + margin: 0 auto

<style type="text/css">
  .parent {
    width: 100%;
    height: 100px;
    background: #ccc;
  }
  .child {
    width: 100px; /*必须定宽*/
    height: 100px;
    background: #c9394a;
    display: block;
    margin: 0 auto;
  }
</style>

<div class="parent">
    <div class="child">水平居中</div>
</div>


● 优点：只需要对子级元素进行设置就可以实现水平方向居中布局效果

● 缺点：如果子级元素脱离文档流（float 或 absolute 或 fixed），会导致 margin 属性的值无效


3、多个块级元素

父级 text-align + 子级 display: inline-block

<style type="text/css">
  .parent {
    width: 100%;
    height: 100px;
    background: #ccc;
    text-align: center;
  }
  .child {
    width: 100px;
    height: 100px;
    background: #c9394a;
    display: inline-block;
  }
</style>

<div class="parent">
  <div class="child">水平居中</div>
</div>


● 优点：浏览器兼容性比较好（IE6~9）

● 缺点：text-align 属性具有继承性，导致子级元素的文本也是居中显示的（见上图中的文字“水平居中”四个字），块级改为inline-block换行、空格会产生元素间隔


4、父级 position: relative + 子级 position: absolute + transform （IE 9+）

relative、absolute、fixed 这三个值都是开启定位，position 的默认值 static 表示不开启定位

relative 是不脱离文档流的，absolute、fixed 都是脱离文档流的

原理

● left 属性：50%，设置子级元素相对于父级元素左边50%

● transform 属性：子级元素往左（往左是负值）移动它自身宽度的一半

<style type="text/css">
  .parent {
    width: 100%;
    height: 100px;
    background: #ccc;
    position: relative;
  }
  .child {
    width: 100px;
    height: 100px;
    background: #c9394a;
    position: absolute;
    left: 50%;
    transform: translateX(-50%); /*自身宽度一半,等同于margin-left: -50px;*/
  }
</style>

<div class="parent">
  <div class="child">水平居中</div>
</div>


● 优点：无论父级元素是否脱离文档流，不影响子级元素水平居中效果（子级元素设置了绝对定位，父级元素开启了定位即已经脱离了文档流）

● 缺点：margin-left 需要知道宽度值，transform 属性是CSS3中新增属性，有浏览器兼容性问题（IE9+）


5、父级：display: flex + justify-content: center

<style type="text/css">
  .parent {
    width: 100%;
    height: 100px;
    background: #ccc;
    display: flex;
    justify-content: center; 
  }
  .child {
    width: 100px;
    height: 100px;
    background: #c9394a;
  }
</style>

<div class="parent">
  <div class="child">水平居中</div>
</div>



移动端在兼容性允许的情况下优先考虑flex

● 优点：灵活强大，简单方便，容易理解

● 缺点：PC端兼容性不好，移动端（Android4.0+）


小结：

● 对于水平居中，我们应该先考虑，哪些元素有自带的居中效果，最先想到的应该就是 text-align:center 了，但是这个只对行内内容有效，

   所以我们要使用 text-align:center 就必须将子元素设置为 display: inline; 或者 display: inline-block; 

● 其次就是考虑能不能用margin: 0 auto; ，因为这都是一两句代码能搞定的事，实在不行就是用绝对定位去实现了。

● 移动端能用flex就用flex，简单方便，灵活并且功能强大，无愧为网页布局的一大利器！

水平居中 text-align => margin: 0 auto => position: absolute => display: flex

水平垂直居中布局

1、行内/行内块级/图片

<style type="text/css">
    .parent {
        background: #e5edff;
        width: 100%;
        height: 200px;
        text-align: center;

    <img src="3.jpg"/>
</div>


2、父级 relative 子级 absolute + top + left + margin 负边距（已知水平垂直居中元素的宽高）

需要知道被水平垂直居中元素的高和宽，才能计算出margin值，兼容所有浏览器。

        background: #ccc;
        width: 200px;

    居中
    <div class="child"></div>
</div>


3、父级 relative 子级 absolute + top: 0、bottom: 0、left: 0、right: 0 + margin: auto （未知父子元素的宽高，不定宽高）

原理：

当top、bottom为0时,margin-top&bottom设置auto的话会无限延伸占满空间并且平分；当left、right为0时,margin-left&right设置auto的话会无限延伸占满空间并且平分


<style type="text/css">
    .parent {
        background: #ccc;
        width: 200px;
        height: 200px;
        position: relative;
    }
    .child {
        background: #c9394a;
        width: 100px;
        height: 100px;
        position: absolute;
        margin: auto;
        top: 0;
        bottom: 0;
        left: 0;
        right: 0;
    }
</style>

<div class="parent">
    居中
    <div class="child"></div>
</div>


● 优点：无需关注宽高；兼容性较好(IE8+)

● 缺点：代码较多；脱离文档流

4、父级 relative 子级 absolute + top + left + transform（未知父子元素的宽高，不定宽高）

<style type="text/css">

        background: #ccc;
        height: 200px;
        width: 200px;
        position: relative;
    }

        width: 100px;
        height: 100px;
        position: absolute;
        top: 50%;
        left: 50%;
        transform: translate(-50%, -50%); /*定宽高时等同于margin-left:负自身宽度一半;margin-top:负自身高度一半;*/

    }
</style>

<div class="parent">
    居中
    <div class="child"></div>
</div>


● 优点：使用margin兼容性好；不管是块级还是行内元素都可以实现

● 缺点：代码较多；脱离文档流；使用margin需要知道宽高；使用transform兼容性不好（IE9+）

<div class="parent">
    居中

宽高不定；浏览器兼容性比较好（IE8+）

● 缺点：

vertical-align 属性具有继承性，导致父级元素的文本也是居中显示的（见上图中的文字“垂直居中”四个字）

7、浮动元素的水平垂直居中

<div class="wrap">  
  <div class="father">
    <div class="child">居中居中居中</div>
  </div>
</div>

.wrap {
  float: left;
  width: 30%;
  height: 400px;
  margin: 10px;
  background: #ccc;
}

.father {
  /* 居中的元素的父元素.ele */
  float: left;  /* 收缩宽度 */
  position: relative;
  left: 50%; /*.ele左边和垂直中线对齐*/
  top: 50%; /* .ele上边和水平中线对齐 */
}  

.child {
  position: relative;
  left: -50%; /* 根据父元素.ele的50%宽度往左拉 */
  padding: 10px;
  transform : translate(0, -50%); /* 根据自身(.ele-inner)的高度(等于.ele高度)拉上去50% */
  background: #333;
  color: #fff;
}


小结：

● 一般情况下，水平垂直居中，我们最常用的就是绝对定位加负边距了，缺点就是需要知道宽高，使用 transform 倒是可以不需要，但是兼容性不好（ie9+）；

● 其次就是绝对居中，绝对定位设置 top、left、right、bottom 为 0，然后 margin:auto;  让浏览器自动平分边距以达到水平垂直居中的目的；

● 如果是行内/行内块级/图片这些内容，可以优先考虑 line-height 和v ertical-align 结合使用，不要忘了还有 text-align 

   这个方法代码其实不多，就是理解原理有点困难，想要熟练应对各种情况还需好好研究；

● 移动端兼容性允许的情况下能用 flex 就用 flex

line-height + text-align  + vertical-align

=> 父级 relative 子级 absolute + top: 50% + left: 50% + margin 负边距

=> 父级 relative 子级 absolute + top: 0、bottom: 0、left: 0、right: 0 + margin: auto  

=> 父级 relative 子级 absolute + top: 50% + left: 50% + transform: translate(-50%, -50%)

=>flex 父级 flex + justify-content + align-items 
 
=>float 父级 relative + top: 50% + left: 50% 子级 relative + left: -50% + transform: translate(0, -50%)

三列布局

经典三列布局，也叫做圣杯布局【Holy Grail of Layouts】是Kevin Cornell在2006年提出的一个布局模型概念

在国内最早是由淘宝UED的工程师传播开来，在中国也有叫法是双飞翼布局

它的布局要求有几点：

1、三列布局，中间宽度自适应，两边定宽；

2、中间栏要在浏览器中优先展示渲染；

3、允许任意列的高度最高；

4、要求只用一个额外的DIV标签；

5、要求用最简单的CSS、最少的HACK语句；

在不增加额外标签的情况下，圣杯布局已经非常完美，圣杯布局使用了相对定位，以后布局是有局限性的，而且宽度控制要改的地方也多。

在淘宝UED（User Experience Design）探讨下，增加多一个div就可以不用相对布局了，只用到了浮动和负边距，这就是我们所说的双飞翼布局。

需求

圣杯布局左中右三列布局，渲染顺序中间列书写在前保证提前渲染，左右两列定宽，中间列自适应剩余宽度。

圣杯布局与双飞翼布局的不同之处，圣杯布局的的左中右三列容器没有多余子容器存在，通过控制父元素的 padding 空出左右两列的宽度。

关键点

● 圣杯布局的关键点是通过 margin-left 与 left 属性将左右两列放置到准确的位置

● 圣杯布局的关键点父元素需要设置 padding

● 圣杯布局的关键点 margin-left 取值为百分比时，是以其父元素的宽度为基准的

        // 3 求出列数
        var col = parseInt(boxW / itemW);
        // 4 求出间距（图片与图片之间右边和上边的距离）
        var distance = (boxW - itemW * col) / (col - 1);
       
        for (var i = 0; i < items.length; i++) {
            // 第一行图片布局 
            if (i < col) {
                items[i].style.left = (itemW + distance) * i + 'px';
                // 存第一行所有图片的高度  
                arr[i] = items[i].offsetHeight;
            }
            // 从第二行开始每张图片都找剩下最矮的那一列的下面放
            else {
                // 找到最矮那一列图片的高度和索引  
                var minH = getMin(arr).minH;
                var minI = getMin(arr).minI; 
                // 设置第二行开始每张图片的位置                
                items[i].style.top = minH + distance + 'px';
                items[i].style.left = minI * (itemW + distance) + 'px'; 
                // 更新图片最小高度，让从第二行开始到第20张图片都找最矮的那一列的下面放，而不是重叠在第一行最矮的那一列下方
                arr[minI] = minH + distance + items[i].offsetHeight;
            }
        }
    }
    // 获取数组的最小值以及索引
    function getMin(arr) {
        var obj = {};
        obj.minH = arr[0];
        obj.minI = 0;
        for (var i = 0; i < arr.length; i++) {
            if (obj.minH > arr[i]) {
                obj.minH = arr[i];
                obj.minI = i;
            }
        }
        return obj;
    }

function addEventListener(element, eventName, fn) {
    // 判断当前浏览器是否支持addEventListener 方法
    if (element.addEventListener) {
        element.addEventListener(eventName, fn); // 第三个参数 默认是false
    }
    else if (element.attachEvent) {
        element.attachEvent('on' + eventName, fn);
    }
    else {
        // 相当于 element.onclick = fn;
        element['on' + eventName] = fn;
    }
}

function removeEventListener(element, eventName, fn) {
    if (element.removeEventListener) {
        element.removeEventListener(eventName, fn);
    }
    else if (element.detachEvent) {
        element.detachEvent('on' + eventName, fn);
    }
    else {
        element['on' + eventName] = null;
    }
}
