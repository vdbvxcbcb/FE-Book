27-浮动精粹总结

浮动

浮动的特点可以用六点总结：脱标补座（其它补上）、包裹、破坏、贴边、文字重排、与上一个元素底部对齐或顶部对齐!

浮动元素不会超过容器的上padding，即浮动元素的左右外边界不能超出包含块的左右内边界。

浮动非替换元素必须为其指定width，否则元素的width会趋于0而导致浮动元素不能完整显示。

浮动元素互相永不会重叠

float在绝对定位和display为none时不会被应用！

浮动布局的核心原理：包含块（包裹）的概念——需要使用标准流的块级父元素包裹浮动的子元素。避免浮动元素脱离文档流时，其它标准流元素占据浮动元素的位置跑到浮动元素下方！！！

清除浮动，更确切地说是闭合浮动，其实通常就一个目的：就是解决父元素高度塌陷的问题！clear属性是该元素不受其他浮动元素影响，把影响该元素的浮动元素看成块级元素的意思。

:after伪元素法：

.clearfix:after {
    content:''; 
    display:block;
    height:0;
    visibility:hidden;
    clear:both;
}
.clearfix {
    *zoom: 1;
}

双伪元素法：

.clearfix:before, .clearfix:after {   
    display: table;
    content: "";
}
.clearfix:after {
    clear: both;
}
.clearfix {
    *zoom: 1;
}

父元素设置overflow：hidden或auto

父元素设置固定高度


普通流
● 除根元素、浮动元素和绝对定位元素外，其它元素都在普通流之内(in-flow) （注：固定定位已经包含在绝对定位里了）
● 而根元素、 浮动和绝对定位的元素会脱离普通流(out of flow)
● 普通流中的盒子，属于块级格式化上下文或行级格式化上下文

后浮动的元素永不会超过先浮动元素的顶端。

浮动元素会尽可能高地放置，便这个高受限于规则1和规则2。

浮动元素会从文档正常流中删除，但它仍会影响布局。

一旦元素具有了浮动属性，它便成为了一个块级元素。

浮动非替换元素必须为其指定width，否则元素的width会趋于0而导致浮动元素不能完整显示。

浮动元素互相永不会重叠。

清除浮动，一般有clear: both/left/right和overflow: hidden。首选clear，如果目标区块存在溢出元素时，会被overflow: hidden截断。
